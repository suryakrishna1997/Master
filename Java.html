1. Types & Type Coercion

Definition: A "type" is a category that tells JavaScript what kind of value something is — a number, a string, a boolean, or a container of other values. Types exist so the engine knows what operations are valid and how to store the data efficiently.

The 8 Primitive Types

TypetypeofExampleNotes
undefined"undefined"let x;Absence of value; default for uninitialized variables
null"object" (bug)let x = null;Intentional absence; typeof returns "object" for legacy reasons
boolean"boolean"true, falseLogical true/false
number"number"42, 3.14, NaNIEEE 754 double-precision 64-bit float

⚠️ Critical: typeof null === "object"

This is a language bug from the first version of JavaScript that can never be fixed due to backward compatibility. The type tag for objects was 0x00 and null was represented as the NULL pointer (also 0x00).

Analogy: Types Are Like Containers

Imagine organizing a warehouse. You have small envelopes for letters (primitives) and large bins for collections (objects). Primitives are copied when you hand them to someone — they get their own envelope. Objects are shared — if you hand someone a bin, you're both pointing at the same bin. Change the contents, and everyone sees it.

Number Internals: IEEE 754 Double Precision

// The classic floating point problem
0.1 + 0.2 === 0.3  // false — 0.30000000000000004

// Solution 1: Use Number.EPSILON
function isEqual(a, b) {
  return Math.abs(a - b) < Number.EPSILON;
}

// Solution 2: ES2026 Math.sumPrecise
Math.sumPrecise([0.1, 0.2, 0.3])  // 0.6 — exact!
Why This Matters in Real Code

When you write let a = 5; let b = a; b = 10;, a stays 5 because numbers are primitives — copied by value. But when you write let obj1 = {x: 5}; let obj2 = obj1; obj2.x = 10;, obj1.x is now 10 too because both variables point to the same object.

Real-world scenario: You're building a form validator. A user leaves a field empty. Is that undefined (they never touched it) or null (they deliberately cleared it)? Your validation logic might treat these differently.

2. Scope & Closures

Definition: Scope answers the question: "Where can I see and use this variable?" It's like visibility rules in a building — some rooms are private, some are shared, some are public.

Lexical Scope

JavaScript uses lexical (static) scoping. A function's scope is determined by where it is defined, not where it is called.

const x = 10;
function outer() {
  const y = 20;
  function inner() {
    const z = 30;
    return x + y + z;
  }
  return inner;
}
const fn = outer();
fn();  // 60 — inner still has access to y via closure
Analogy: Nested Russian Dolls

Each function is like a Russian doll. The inner doll can see everything in the outer dolls, but the outer dolls can't peek inside.

3. Hoisting & Temporal Dead Zone

Definition: Hoisting is JavaScript's behavior of moving variable and function declarations to the top of their scope during compilation — before any code actually runs.
// What you write:
console.log(x);  // undefined
var x = 5;

// What the engine sees:
var x;           // Declaration hoisted, initialized undefined
console.log(x);  // undefined
x = 5;           // Assignment stays in place
Analogy: The Conference Room Reservation

With var, the room is reserved but empty until you show up. With let and const, the room is locked until the exact reservation time — try to enter early and security throws you out (ReferenceError).

The Temporal Dead Zone (TDZ)

let x = 10;
function foo() {
  // TDZ starts here for inner x
  console.log(x);  // ReferenceError!
  let x = 20;      // TDZ ends here
}

5. Prototypes & Inheritance

Definition: A prototype is a fallback object. When you ask an object for a property it doesn't have, JavaScript looks at that object's prototype and keeps climbing up the chain until it finds the property or reaches null.

};
const dog = { sound: 'Woof!', __proto__: animal };
dog.speak();  // "Woof!"

7. Async Programming

Definition: Asynchronous JavaScript means the code doesn't wait for slow operations to finish before moving on. The event loop is the mechanism that checks for completed tasks and runs their callbacks.

The Event Loop: Microtasks vs Macrotasks

console.log('1');
setTimeout(() => console.log('5'), 0);  // macrotask
Promise.resolve().then(() => console.log('3'));  // microtask
console.log('2');
// Output: 1, 2, 3, 5
Analogy: A Restaurant Kitchen

The chef starts the steak (slow), sets a timer, starts the salad (fast), serves the salad when done. When the steak timer rings, finishes the steak. Single-threaded but managing multiple tasks concurrently.

async/await

async function fetchUser(id) {
  const response = await fetch(`/api/users/${id}`);
  return await response.json();
}

15. Error Handling

Definition: An Error is JavaScript's way of saying "something went wrong." When thrown, the engine stops and looks for a catch block. If none is found, the program crashes.
Error
├── RangeError        // Number out of range
├── ReferenceError    // Invalid reference
├── SyntaxError       // Parse error
├── TypeError         // Value not of expected type
└── AggregateError    // ES2021: Multiple errors

Custom Errors

class ValidationError extends Error {
  constructor(message, field) {
    super(message);
    this.name = 'ValidationError';
    this.field = field;
  }
}
try {
  throw new ValidationError('Invalid email', 'email');
} catch (err) {
  if (err instanceof ValidationError) {
    console.error(`Field ${err.field}: ${err.message}`);
  }
}

18. Performance Optimization

Definition: JavaScript performance is about writing code that matches the engine's assumptions. V8 makes optimizations that break when those assumptions are violated.
// Good: consistent object shape (same hidden class)
function Point(x, y) { this.x = x; this.y = y; }

// Debounce: fire only after inactivity
function debounce(fn, delay) {
  let id;
  return function(...args) {
    clearTimeout(id);
    id = setTimeout(() => fn.apply(this, args), delay);
  };
}

19. V8 Engine Internals

Definition: V8 compiles JavaScript to machine code through multiple stages, each making different trade-offs between startup speed and execution speed.
  1. Ignition — Bytecode interpreter, fast startup
  2. Sparkplug — Quick baseline machine code
  3. TurboFan — Speculative optimization, highly optimized machine code
🔬 TurboFan Optimization

TurboFan uses Sea of Nodes IR and applies inlining, constant folding, dead code elimination, and loop unrolling. If assumptions fail, it deoptimizes back to baseline code.

20. Security Patterns

Definition: JavaScript security is about preventing malicious code from executing and protecting user data — even when interacting with untrusted input or third-party scripts.
// NEVER: innerHTML with user input

// ALWAYS: textContent

// Prototype pollution guard
function safeMerge(target, source) {
  for (const key in source) {
    if (key === '__proto__' || key === 'constructor' || key === 'prototype') continue;
    target[key] = source[key];
  }
}

Definition: Functional programming (FP) is a paradigm where computation is treated as the evaluation of mathematical functions, avoiding changing state and mutable data. In JavaScript, FP emphasizes pure functions, immutability, and function composition.


        
Pure Functions

        
A pure function always returns the same output for the same input and has no side effects.

        
// Impure: depends on external state
let count = 0;
function increment() { return ++count; }

// Pure: same input → same output, no side effects
const add = (a, b) => a + b;
add(2, 3);  // Always 5


        
Currying

        
Transforms a function with multiple arguments into a sequence of functions, each taking a single argument.

        
const multiply = a => b => a * b;
const triple = multiply(3);
triple(4);  // 12
triple(10); // 30

// Real-world: pre-configured API client
const fetchWithBase = baseUrl => endpoint => fetch(baseUrl + endpoint);
const api = fetchWithBase('https://api.example.com');
api('/users');  // fetch('https://api.example.com/users')


        
Function Composition

        
const compose = (...fns) => x => fns.reduceRight((v, f) => f(v), x);
const pipe = (...fns) => x => fns.reduce((v, f) => f(v), x);

const toUpper = s => s.toUpperCase();
const exclaim = s => s + '!';
const shout = pipe(toUpper, exclaim);
shout('hello');  // 'HELLO!'


        
Memoization

        
function memoize(fn) {
  const cache = new Map();
  return function(...args) {
    const key = JSON.stringify(args);
    if (cache.has(key)) return cache.get(key);
    const result = fn.apply(this, args);
    cache.set(key, result);
    return result;
  };
}

const fib = memoize((n) => n <= 1 ? n : fib(n-1) + fib(n-2));
fib(100);  // Instant — without memoization this would hang

        
Analogy: A Recipe with No Surprises
A pure function is like a recipe that uses only the ingredients you give it — no sneaking into the pantry for extra salt. The result is always predictable, testable, and safe to use anywhere.

        


        
26. Design Patterns in JavaScript

        
Definition: Design patterns are reusable solutions to common software design problems. They provide a shared vocabulary and proven approaches to structuring code.


        
Module Pattern

        
const Counter = (function() {
  let count = 0;  // Private
  return {
    increment() { return ++count; },
    decrement() { return --count; },
    get() { return count; }
  };
})();
Counter.increment();  // 1
Counter.count;            // undefined — truly private


        
Singleton Pattern

        
class Database {
  static #instance = null;
  #connection = null;
  constructor() {
    if (Database.#instance) return Database.#instance;
    Database.#instance = this;
  }
  static getInstance() {
    return Database.#instance ??= new Database();
  }
}


        
Factory Pattern

        
class User { constructor(name) { this.name = name; } }
class Admin extends User { constructor(name) { super(name); this.role = 'admin'; } }
class Guest extends User { constructor(name) { super(name); this.role = 'guest'; } }

const userFactory = (type, name) => {
  const types = { user: User, admin: Admin, guest: Guest };
  return new (types[type] || User)(name);
};
userFactory('admin', 'Alice');  // Admin { name: 'Alice', role: 'admin' }


        
Observer Pattern

        
class EventEmitter {
  #listeners = new Map();
  on(event, cb) {
    if (!this.#listeners.has(event)) this.#listeners.set(event, []);
    this.#listeners.get(event).push(cb);
    return () => this.off(event, cb);
  }
  emit(event, data) {
    this.#listeners.get(event)?.forEach(cb => cb(data));
  }
  off(event, cb) {
    const cbs = this.#listeners.get(event) || [];
    this.#listeners.set(event, cbs.filter(c => c !== cb));
  }
}

self.onmessage = function(e) {
  if (e.data.action === 'fib') {
    const result = fibonacci(e.data.n);  // Heavy computation
    self.postMessage(result);
  }
};

// SharedArrayBuffer for high-performance data sharing

const view = new Int32Array(shared);

Due to Spectre vulnerabilities, SharedArrayBuffer requires cross-origin isolation headers (Cross-Origin-Opener-Policy: same-origin and Cross-Origin-Embedder-Policy: require-corp) to be enabled.

// Debounce
function debounce(fn, delay) {
  let timer;
  return function(...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}

// Throttle
function throttle(fn, limit) {
  let inThrottle;
  return function(...args) {
    if (!inThrottle) {
      fn.apply(this, args);
      inThrottle = true;
      setTimeout(() => inThrottle = false, limit);
    }
  };
}

// Deep Clone
function deepClone(obj, hash = new WeakMap()) {
  if (Object(obj) !== obj) return obj;
  if (hash.has(obj)) return hash.get(obj);
  const result = Array.isArray(obj) ? [] : {};
  hash.set(obj, result);
  return Object.assign(result, ...Object.keys(obj).map(key => ({
    [key]: deepClone(obj[key], hash)
  })));
}

// Flatten Array
const flatten = arr => arr.reduce((flat, item) =>
  flat.concat(Array.isArray(item) ? flatten(item) : item), []);

// Promise.all with concurrency limit
async function asyncPool(concurrency, iterable, iteratorFn) {
  const ret = [];
  const executing = new Set();
  for (const item of iterable) {
    const p = Promise.resolve().then(() => iteratorFn(item));
    ret.push(p);
    executing.add(p);
    const clean = () => executing.delete(p);
    p.then(clean, clean);
    if (executing.size >= concurrency) await Promise.race(executing);
  }
  return Promise.all(ret);
}
